### Iteration 2 (2026-04-12)

**Starting state**: 33/39 Spout=MATCH

**Root causes identified and fixed**:
1. **Bare passive model names** (cap→c, res→r): writeSubcircuitFromPR was emitting default model names like "cap"/"res" for bare R/C/L devices. NETGEN treats these as different device classes than bare "c"/"r". Fix: added isPassiveDevice() check — skip default model for passives with empty model_name.
2. **MOSFET W=0 property errors**: Circuits with nfin= param (no explicit w=) got W=0.0000e0 in output, causing NETGEN property mismatch. Fix: writeMosfetParams() skips W= and L= when value is 0.
3. **Single-subcircuit X device terminals**: SpiceWriter path used gate/source/drain/body terminal order for X devices, but X device pins have .anode terminal type → all "?". Fix: always use writeSubcircuitFromPR for all circuits (single and multi-subcircuit).
4. **Unclosed subcircuit (missing .ends)**: Files ending with .END instead of .ends left device_end=0 → empty device range. Fix: tokenizer finalize() closes unclosed subcircuits.

**Files changed**:
- `src/lib.zig`: Removed single-subcircuit SpiceWriter path, always use writeSubcircuitFromPR; added isPassiveDevice(), writeMosfetParams()
- `src/netlist/tokenizer.zig`: finalize() closes unclosed subcircuits

**Verified**: 38/39 MATCH (was 33/39). Fixed: align_sc_dc_dc_converter, align_switched_capacitor_filter, align_vco_dtype12_hierarchical_res, align_vco_dtype12_hierarchical_res_constrained, jku_sar_adc_wrapper. No regressions.
