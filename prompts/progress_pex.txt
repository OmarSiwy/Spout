### Root Cause Analysis (refined)

**TWO independent problems:**

1. **Substrate count gap** (folded_cascode -5, sar_adc -2):
   The ≥3 merged-net heuristic in pex.zig incorrectly absorbs internal signal nets
   into SUBSTRATE_NET. For folded_cascode: 15 raw unique pin nets, but only 7
   classified as signal (27 of 44 pins mapped to substrate). Body-terminal detection
   shows only 6 pins should be body/substrate.

2. **Coupling count gap** (diff_pair +5, five_transistor_ota +4, folded_cascode -5, sar_adc -3):
   Device-pin coupling model couples ALL pin pairs on different nets within 15µm.
   Too aggressive for small circuits, too limited for large ones (because missing
   substrate nets can't have coupling pairs).

### Experiments Tried

**A. Remove ≥3 heuristic + body-terminal cap rewriting:**
- Correctly identifies VDD/VSS via body terminal pins (ctx.pins.terminal == .body)
- diff_pair improved: 21→23 (CLOSE)
- folded_cascode improved: 15→18
- sar_adc improved: 11→13
- BUT current_mirror regressed: 6→3 (lost VDD/VSS as proxy for internal nodes)
- five_transistor_ota unchanged: 10

**B. Remove ≥3 heuristic + body-terminal merge-map only (no cap rewrite):**
- VDD/VSS routing caps preserved (act as internal-node proxies)
- diff_pair MATCH: 22/23 (0.957x) ← BEST RESULT
- folded_cascode: 18/25 (improved)
- sar_adc: 13/16 (improved)
- current_mirror regressed: 4/6 (coupling model skips body nets)
- five_transistor_ota unchanged: 10/5

### Key Insights

1. VDD/VSS routing-derived caps accidentally compensate for Magic's internal
   diffusion nodes (a_xxxxx#) that Spout can't model. Absorbing VDD/VSS fixes
   large circuits but breaks small ones.

2. The ≥3 heuristic works for current_mirror/diff_pair (doesn't trigger) but
   is too aggressive for folded_cascode (absorbs 8 signal nets with VDD/VSS).

3. Body-terminal detection (ctx.pins.terminal == .body) correctly identifies
   VDD/VSS but we can't simply absorb them without also modelling internal
   diffusion nodes to compensate.

## Iteration 3 — Diffusion Node Model (2026-04-12)

### Changes Made
1. **types.zig**: Added `body_net_ids: ?[]const u32 = null` field to PexConfig
2. **lib.zig**: Collects body net IDs from pin terminal data before extractFromRoutes
3. **lib.zig**: Added internal diffusion node model after device-pin coupling:
   - Scans S/D pins on body-terminal nets (VDD/VSS)
   - Uses pin-level proximity (5µm) + UF clustering to detect shared junctions
   - Creates virtual substrate cap per cluster + gate-only coupling caps
4. **pex.zig**: ≥3 heuristic retained (removing it helps folded_cascode/sar_adc
   but breaks five_transistor_ota; body-net-in-UF approach failed because UF
   merges signal nets with body nets via cross-layer via shorts)

### Experiments Tried (iteration 3)

**A. Remove ≥3 heuristic entirely (no body-net in pex.zig):**
- folded_cascode improved: 11→18/25 (0.720x)
- sar_adc improved: 8→17/16 (1.062x, almost MATCH!)
- five_transistor_ota regressed: 6→10/5 (2.0x) — extra signal nets from route segments
- diff_pair unchanged: 22/23

**B. Body-net propagation via UF in pex.zig:**
- FAILED: UF cross-layer via-point detection merges signal nets with body nets,
  causing over-absorption. current_mirror dropped to 3/6.

**C. Diffusion nodes with device-center distance (5µm) + all-signal coupling:**
- current_mirror: +1 node, +3 caps → 7/6 (overshoot +1)
- diff_pair: +2 nodes, +11 caps → 33/23 (massive overshoot)
- five_transistor_ota: +4 nodes, +10 caps → 16/5 (massive overshoot)
- folded_cascode: 0 nodes (threshold too tight)

**D. Diffusion nodes with pin-proximity (5µm) + body-only + gate-only coupling:**
- current_mirror: +1 node, +2 caps → 6/6 (MATCH!)
- diff_pair: 0 nodes → 22/23 (preserved MATCH)
- five_transistor_ota: 0 nodes → 6/5 (unchanged)
- folded_cascode: 0 nodes → 11/25 (threshold still too tight for this layout)
- sar_adc: +2 nodes, +3 caps → 11/16 (improved from 8)

### Key Insights (iteration 3)
1. ≥3 heuristic is essential for five_transistor_ota (absorbs VDD/VSS-shorted route
   nets correctly). Removing it releases ~4 extra signal nets → overcounting.
2. Body-net propagation through pex.zig's UF is unreliable because the UF merges
   signal and body nets through via-point short detection.
3. Pin-proximity diffusion model works perfectly for current_mirror (6/6 MATCH)
   but doesn't trigger for folded_cascode because body-net S/D pins are >5µm apart.
4. Gate-only coupling prevents overcounting (vs all-signal coupling).

### Remaining Problems (priority order)

1. **folded_cascode (11 vs 25, -14)**: ≥3 heuristic absorbs internal signal nets.
   Need to either: (a) increase diffusion node threshold significantly (>10µm) but
   risk overcounting on smaller circuits, OR (b) use topology-based model (count
   devices with S/D on body nets → create N-1 virtual nodes, no proximity check),
   OR (c) post-process pex.zig's SUBSTRATE_NET mapping in lib.zig to restore
   incorrectly absorbed signal nets using body-terminal data.

2. **sar_adc (11 vs 16, -5)**: Same root cause as folded_cascode. Partially helped
   by diffusion model but ≥3 heuristic still absorbs signal nets.

3. **five_transistor_ota (6 vs 5, +1)**: Device-pin coupling creates 1 extra pair.
   Fix: reduce inter-device coupling distance from 15µm, or restrict coupling to
   same-device gate↔drain/source only.

### Recommended Next Step
**Option (c) for folded_cascode/sar_adc**: In lib.zig after extractFromRoutes,
scan pex.zig's merge_from/merge_to for nets mapped to SUBSTRATE_NET. Check each
against body-terminal nets. For non-body nets that were absorbed by the ≥3 heuristic:
restore them as signal nets by re-adding substrate caps. This fixes folded_cascode
and sar_adc without affecting five_transistor_ota (which correctly uses the heuristic).
