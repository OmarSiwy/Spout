### Root Cause Analysis

**Magic R=0**: Magic ext2spice without `extract do resistance` produces 0 R.
Adding `extract do resistance` causes segfault. Script fixed to skip R check when Magic R=0.

**Magic C structure** (per circuit from .ext files):
- C count = #substrate_nets + #coupling_pairs
- current_mirror: 3 substrate + 3 coupling = 6
- diff_pair: 7 substrate + 16 coupling = 23
- five_transistor_ota: 3 substrate + 2 coupling = 5
- folded_cascode: 12 substrate + 13 coupling = 25
- sar_adc_comparator: 7 substrate + 9 coupling = 16

**Why coupling is missing in Spout**:
Magic's coupling comes from tile-plane adjacency including device terminal regions
(diffusion, poly, LI pads). RouteArrays don't have device terminal geometry.
Sidewall coupling in pex.zig only finds parallel routing segments on same layer.
Interlayer overlap cap added but is too aggressive (counts device stubs).

**Why substrate count differs**:
- Device-pin loop in lib.zig adds substrate caps for nets without routes → over-counts
- Without it, many nets missing (current_mirror 2 instead of 3)
- five_transistor_ota always over-counts (extra route segments on device-internal nets)

### Key Insight — What's Needed
The fundamental gap: Spout extracts from RouteArrays (routing wire segments only).
Magic extracts from the full GDS layout (including device terminal regions, poly,
diffusion areas). Magic's coupling caps come from:
1. Inter-layer overlap between ANY regions on different layers (not just routing)
2. Sidewall coupling between ALL adjacent regions (including device pads)

To fully match Magic, Spout would need one of:
A. Extract from the full GDS tile planes (like Magic does) — major architectural change
B. Add device-proximity coupling in lib.zig using pin positions and device geometry
C. Tune the overlap/coupling parameters to produce the right COUNT per circuit

Option B seems most tractable: compute coupling between device pins on different nets
based on their physical separation, using the device positions available in lib.zig.
For each pair of nearby pins on different nets, emit a coupling C element proportional
to a simple proximity model. This would add the ~3-16 missing coupling elements.

### Next Steps
1. In lib.zig post-processing: add device-pin proximity coupling
   - For each pair of device pins on different nets within ~2µm
   - Compute coupling C from pin-to-pin distance using overlap_cap coefficients
   - This fills the coupling gap for simple circuits
2. Tune min_routing_len filter to reduce five_transistor_ota overcount
3. Or remove overlap from pex.zig entirely and rely on device-pin coupling only
